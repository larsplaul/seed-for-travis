Semester Seed - REST Backend

This is a backend for this matching React Front End


This Backend provides, out of the box:

  • Token Based Authentication
  • Two predefined user roles (with a sample REST-endpoint for each)
    • User
    • Admin
  • It uses an in-memory database, for Integration/Unit-tests that requires a database, so you can run tests right after cloning.

    It ships with a persistence.xml file that contains three persistence units as outlined below:

    • pu_development: You MUST change this to point to your own local MySQL database, before you run the project with the front-end. Run the file utils.makeTestUsers before you execute the project to set up the test users.
    • pu_memorydb_mock: This is the persistenc unit used by the default tests. It uses an in-memory Derby database.
    • pu_mySql_travis_Integration: This is a persistenc unit meant for REAL integration tests (that is using the right database) on Travis

  • Built in filters to allow for b CORS


Using the two projects

  • For development just start this server, and start the client using npm start (in this order) and it should work
  • For deployment:
    • either deploy on separate servers (rememember to set serverURL in the clients package.json)
    • or add everything from the clients build folder (after a npm run build) to the root of this projects Web Pages folder.
      Read the instructions related to server root after the npm run build on the client (in the terminal)
